# Gitの運用

## ブランチの作り方

ブランチの作り方は以下の通り

```bash
git branch <ブランチ名>
```

作られたかどうかを確認したい場合は以下のコマンドを使う。

```bash
git branch
```

このコマンドによって、今存在するbranchが一覧で出てくる。

### (おまけ) ブランチ名の付け方

そのブランチで実装することをブランチ名につけることが多い。  
例 :

- LIFモデルを実装する -> feature/LIF
- PQNモデルのミスを修正する -> fix/PQN

特に、複数人で作業する可能性があるブランチはこのようなルールでつけた方が良い。詳しくはgit-flowモデルで検索。
ただ、この作業は自分しか作業しないとわかっているなら自分の名前などをつけて良い（推奨はされない）。

## ブランチへの移動方法

ブランチの移動方法

```bash
git checkout <ブランチ名> # 古い方法
# or
git switch <ブランチ名> # 現在の方法
```

どちらの方法でも移動できる。昔は`checkout`しかなかったが、いろいろわかりにくい（意味が直感的ではない、一つのコマンドでできることが多すぎるなど）ということで、現在ブランチの切り替えは`switch`コマンドを使うことが多い。VS CodeのGUIでcheckoutという文言があるのはこれが由来。

### (おまけ) GUIとCUI

GUIとCUIが何を指しているかの解説

- GUI (Graphical User Interface) : いわゆる我々が思いつくのUIのこと。つまり、画面上のボタンやアイコンのなどで操作できる。直感的に操作できる。
- CUI (Character User Interface) : いわゆる黒い画面で操作すること。つまり、コマンドなどで操作できる。わかりにくいが、より様々なことができる。

## 作業内容の保存・同期方法

### （前提知識） GitとGitHubの違い

Gitはバージョン管理システム。GitHubはコードの保存・共有システム。  
Gitだけでも動くが、Gitでは個人のバージョン管理になる。  
GitHubを使うことで、複数人のGitプロジェクトを同期できる。

### 作業内容の保存 （ローカルで）

`add`と`commit`というコマンドからなる。

- `add` : 保存する内容を登録する。このことをステージングという。
- `commit` : ステージングしたものを保存する。`-m`をつけることで、どういう内容を保存したかメモできる。

例 : srcにnew.pyファイルを作ったとき

```bash
git branch --show-current # 自分が今いるブランチの確認できる
git add src/new.py
git commit -m "new.pyを作成した"
```

このコマンドは、ローカル（自分のPC）で保存するコマンドであり、GitHubやそれを通した他の人のPCには影響がない。よって、キリが良いタイミングで随時実施する。作業した内容をラベリングできる（`-m`で書く内容ができる）時点でcommitまでするべし。commit自体は割と自由に編集・削除できるので気兼ねなくしましょう。編集方法は各自AIにでも聞いてみてください。失敗してもローカルなので迷惑かかりません。ただし、一度リモート同期したら、変更するのはめんどくさいです。

また、この保存した単位のこともcommitと言う。  
例：昨日のcommitで〇〇を修正したので確認よろしく  
GitHub上ではこのcommitごとに出る。  
[参考：このプロジェクトのCommit履歴](https://github.com/ryutatsu55/SNN_sim/commits/master/)

### 作業内容の同期 (リモートと同期)

ある程度commitできたら、GitHub（=リモート）へ送信して同期する。これをpushと言う。pushしたら他人にも影響するので、他人が操作しているブランチでのpushは気をつけてください。pushした後にpushしたものを変更するのはいろいろだるいので気をつけてください。自分しか操作していないブランチなら割となんとかなりますが。逆にいえば、メインのブランチである`master`にpushするのは超慎重に。

なお、commitとpushは一対一の関係ではない。複数のcommitをまとめてpushできる。

#### まだ存在しないブランチを作成して初めてのpush

```bash
git push -u origin <作成したブランチ名>
```

これをすることで、GitHub上でブランチが作られ、そのブランチで行ったcommitが同期される。

#### すでに存在するブランチへのpush

```bash
git push
```

これだけ。

#### 他の人がcommitしたものをローカルと同期する

他の人がcommitしたものを、自分のPCに反映することをpullと言う。

```bash
git pull
```

他の人によって新しく作られたブランチもこれで反映される。なお、ブランチを移動した場合は、その都度`git pull`した方が良い。

## （おまけ） ブランチを作る意味

以下の意味がある

- 編集が競合しないようにするため
- 作業単位ごとにレビューするため
  - ある程度作業できたら、「このブランチでの変更をメインのブランチ（今回のプロジェクトだと`master`）に取り込んで！」ということができる。その際にこのブランチでの変更をまとめて見ることができるので便利。
